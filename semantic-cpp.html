О докладчике

Константин Владимиров — специалист компании Syntacore. Он работает над компиляторами с 2010 года и в настоящий момент занимается компилятором для архитектуры RISC-V.

О чём доклад

Доклад посвящён тому, что в языке C++ важно различать синтаксис и семантику. Синтаксис — это форма записи конструкций языка. Например, оператор if всегда распознаётся как конструкция ветвления.

Семантика сложнее: в этот момент компилятор не просто читает текст программы, а выполняет определённые действия. Ему нужно понять, какое именно имя используется, какой тип должен быть выведен, нужно ли инстанцировать шаблон и можно ли вычислить выражение во время компиляции.

Главная идея доклада состоит в том, что такие действия можно рассматривать как отдельные семантические процессы, которые взаимодействуют между собой.

Какие процессы рассматриваются

В докладе рассматриваются основные процессы, которые происходят внутри компилятора:

  • разрешение имён;
  • инстанцирование шаблонов;
  • вывод типов;
  • вычисление константных выражений.

Это важно, потому что в C++ многие возможности языка тесно связаны между собой. Иногда для понимания одной конструкции компилятору сначала приходится выполнить несколько других шагов.

Поэтому C++ выглядит сложным не только из-за большого синтаксиса, но и из-за сложных правил смысловой обработки программы.

Примеры кода

Пример простой конструкции, где компилятор распознаёт синтаксис:

if (x > 0) {
    std::cout << "Положительное число";
}

Пример, где компилятору нужно вывести тип:

auto x = 10;
auto y = 3.14;

Пример шаблона, где возникает инстанцирование:

template<typename T>
T sum(T a, T b) {
    return a + b;
}

int result = sum(2, 3);

Пример константного выражения:

constexpr int square(int x) {
    return x * x;
}

constexpr int value = square(5);

Вывод

Из доклада можно сделать вывод, что C++ — это язык, в котором смысл программы формируется не одним действием, а целой системой взаимосвязанных процессов. Чтобы лучше понимать язык, полезно смотреть не только на то, как код записан, но и на то, что именно в этот момент должен сделать компилятор.

Такой взгляд помогает лучше понимать шаблоны, типы, constexpr и устройство компиляции программы.
